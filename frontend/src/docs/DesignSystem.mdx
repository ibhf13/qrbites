# QrBites Design System

The QrBites design system provides a comprehensive set of UI components and design tokens to build consistent, responsive, and accessible user interfaces.

## Color Palette

Our color system is built around primary, secondary, accent, and neutral colors with 11 tonal values each (50-950).

```jsx
// Primary Color - Blue
primary: {
  50: '#f0f9ff',  // Lightest
  100: '#e0f2fe',
  200: '#bae6fd',
  300: '#7dd3fc',
  400: '#38bdf8',
  500: '#0ea5e9', // Base color
  600: '#0284c7',
  700: '#0369a1',
  800: '#075985',
  900: '#0c4a6e',
  950: '#082f49'  // Darkest
}

// Secondary Color - Purple
secondary: {/* Similar structure */}

// Accent Color - Amber
accent: {/* Similar structure */}

// Neutral Color - Gray
neutral: {/* Similar structure */}
```

## Typography

We use a type system that combines Inter (sans-serif) for body text and Poppins (sans-serif) for display text.

```jsx
typography: {
  fontFamily: {
    sans: ['Inter', 'sans-serif'],
    display: ['Poppins', 'sans-serif'],
  },
  fontSize: {
    xs: ['0.75rem', { lineHeight: '1rem' }],
    sm: ['0.875rem', { lineHeight: '1.25rem' }],
    base: ['1rem', { lineHeight: '1.5rem' }],
    lg: ['1.125rem', { lineHeight: '1.75rem' }],
    xl: ['1.25rem', { lineHeight: '1.75rem' }],
    '2xl': ['1.5rem', { lineHeight: '2rem' }],
    '3xl': ['1.875rem', { lineHeight: '2.25rem' }],
    '4xl': ['2.25rem', { lineHeight: '2.5rem' }],
    '5xl': ['3rem', { lineHeight: '1' }],
  },
}
```

## Components

### Typography Component

The Typography component provides consistent text styling across the application.

```jsx
<Typography variant="h1">Heading 1</Typography>
<Typography variant="h2">Heading 2</Typography>
<Typography variant="body1">Regular paragraph text</Typography>
<Typography variant="caption" color="secondary">Caption text with secondary color</Typography>
```

### Button Component

Buttons are used for actions and navigation.

```jsx
<Button>Default Button</Button>
<Button variant="secondary" size="lg">Large Secondary Button</Button>
<Button variant="outline" isFullWidth>Full Width Outline Button</Button>
<Button variant="link">Link Button</Button>
<Button isLoading>Loading Button</Button>
```

### Badge Component

Badges are used to highlight status, count, or category information.

```jsx
<Badge label="New" />
<Badge label="Completed" color="success" variant="light" />
<Badge label="Warning" color="warning" variant="outlined" />
```

### Card Component

Cards are used to group related content.

```jsx
<Card>
  <Card.Header>Card Title</Card.Header>
  <Card.Body>Card content goes here</Card.Body>
  <Card.Footer>Card footer with actions</Card.Footer>
</Card>
```

### Grid System

Our responsive grid system is based on a 12-column layout.

```jsx
<Container>
  <Row>
    <Col span={6} md={4}>Column 1</Col>
    <Col span={6} md={4}>Column 2</Col>
    <Col span={12} md={4}>Column 3</Col>
  </Row>
</Container>
```

### Skeleton Loading

Use skeleton components for loading states.

```jsx
<Skeleton variant="text" height="20px" />
<Skeleton variant="circular" width="40px" height="40px" />
<Skeleton variant="rectangular" height="200px" />
```

## Dark Mode

All components support dark mode through Tailwind's dark mode class. 

```jsx
// Toggle between light and dark mode
<ThemeToggle />
```

## Responsive Design

All components are designed to be responsive following these breakpoints:

- Mobile: < 640px
- Tablet: 640px - 1024px
- Desktop: > 1024px

## Accessibility

Our components follow WCAG 2.1 AA standards and include:

- Proper color contrast
- Keyboard navigation support
- Screen reader compatibility
- Focus management
- Aria attributes
