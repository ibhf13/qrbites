## Introduction

The QrBites design system is built with React, TypeScript, and Tailwind CSS. It provides a set of reusable components, design tokens, and patterns that ensure consistency across the application.

To see the components in action, visit the [Design System Demo](/design-system-demo) page.

## Design Tokens

### Color Palette

Our color system includes primary, secondary, accent, and neutral colors. Each color has 11 tonal values (50-950).

```jsx
// Example usage in components
<div className="bg-primary-500 text-white">Primary Background</div>
<div className="text-secondary-700">Secondary Text</div>
<div className="border-accent-300">Accent Border</div>
```

### Typography

The typography system uses Inter for body text and Poppins for display text.

```jsx
// Example typography usage
<Typography variant="h1">Heading 1</Typography>
<Typography variant="body1">Regular body text</Typography>
<Typography variant="caption" color="secondary">Secondary caption</Typography>
```

### Spacing

The spacing system uses a consistent scale based on multiples of 4px.

```jsx
// Example spacing usage
<div className="p-4">Padding 16px</div>
<div className="m-6">Margin 24px</div>
<div className="gap-2">Gap 8px</div>
```

## Components

### Typography Component

The Typography component provides consistent text styling with various variants.

```jsx
import { Typography } from '../components/common';

// Basic usage
<Typography variant="h1">Heading 1</Typography>
<Typography variant="body1">Regular text</Typography>

// With color
<Typography variant="h2" color="primary">Primary Heading</Typography>

// With alignment
<Typography variant="body1" align="center">Centered text</Typography>

// With gutterBottom (margin-bottom)
<Typography variant="h3" gutterBottom>Heading with margin bottom</Typography>

// With noWrap (prevents text wrapping)
<Typography variant="subtitle1" noWrap>This text will not wrap and instead will truncate with ellipsis</Typography>
```

**Props:**
- `variant`: 'h1' | 'h2' | 'h3' | 'h4' | 'h5' | 'h6' | 'subtitle1' | 'subtitle2' | 'body1' | 'body2' | 'caption' | 'overline'
- `color`: 'primary' | 'secondary' | 'accent' | 'success' | 'warning' | 'error' | 'info' | 'default'
- `align`: 'left' | 'center' | 'right'
- `gutterBottom`: boolean - Adds margin bottom
- `noWrap`: boolean - Prevents text wrapping
- `className`: string - Additional CSS classes

### Button Component

The Button component is used for actions and navigation.

```jsx
import { Button } from '../components/common';

// Basic usage
<Button>Click me</Button>

// Variants
<Button variant="primary">Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="accent">Accent</Button>
<Button variant="outline">Outline</Button>
<Button variant="ghost">Ghost</Button>
<Button variant="link">Link</Button>

// Sizes
<Button size="xs">Extra Small</Button>
<Button size="sm">Small</Button>
<Button size="md">Medium</Button>
<Button size="lg">Large</Button>
<Button size="xl">Extra Large</Button>

// Full width button
<Button isFullWidth>Full Width Button</Button>

// Loading state
<Button isLoading>Loading</Button>

// With icons
<Button leftIcon={<Icon />}>With Left Icon</Button>
<Button rightIcon={<Icon />}>With Right Icon</Button>
```

**Props:**
- `variant`: 'primary' | 'secondary' | 'accent' | 'outline' | 'ghost' | 'link'
- `size`: 'xs' | 'sm' | 'md' | 'lg' | 'xl'
- `isFullWidth`: boolean
- `isLoading`: boolean
- `leftIcon`: React.ReactNode
- `rightIcon`: React.ReactNode
- `className`: string
- All standard button attributes

### Grid System

The grid system provides responsive layout components.

```jsx
import { Container, Row, Col } from '../components/common';

// Basic usage
<Container>
  <Row>
    <Col span={6}>Left column</Col>
    <Col span={6}>Right column</Col>
  </Row>
</Container>

// Responsive columns
<Container>
  <Row>
    <Col span={12} md={6} lg={4}>Column 1</Col>
    <Col span={12} md={6} lg={4}>Column 2</Col>
    <Col span={12} md={12} lg={4}>Column 3</Col>
  </Row>
</Container>

// With spacing
<Container>
  <Row spacing={4}>
    <Col span={6}>Column with spacing</Col>
    <Col span={6}>Column with spacing</Col>
  </Row>
</Container>

// Alignment
<Container>
  <Row justifyContent="center" alignItems="center">
    <Col span={6}>Centered column</Col>
  </Row>
</Container>
```

**Container Props:**
- `maxWidth`: 'xs' | 'sm' | 'md' | 'lg' | 'xl' | '2xl' | 'full' | 'none'
- `padding`: boolean
- `className`: string

**Row Props:**
- `spacing`: 0 | 1 | 2 | 3 | 4 | 5 | 6 | 8 | 10 | 12
- `justifyContent`: 'start' | 'end' | 'center' | 'between' | 'around' | 'evenly'
- `alignItems`: 'start' | 'end' | 'center' | 'baseline' | 'stretch'
- `noWrap`: boolean
- `className`: string

**Col Props:**
- `span`: 1-12 | 'auto' | 'full'
- `sm`, `md`, `lg`, `xl`: Same as span
- `offset`: 0-11
- `order`: 1-12 | 'first' | 'last' | 'none'
- `className`: string

## Theme Support

The design system supports both light and dark themes.

```jsx
import { ThemeToggle } from '../components/common';

// Adding theme toggle button
<ThemeToggle />
```

To access the current theme or toggle the theme programmatically:

```jsx
import { useTheme } from '../contexts/ThemeContext';

const MyComponent = () => {
  const { theme, toggleTheme } = useTheme();
  
  return (
    <div>
      <p>Current theme: {theme}</p>
      <button onClick={toggleTheme}>Toggle Theme</button>
    </div>
  );
};
```

## Responsive Design

The design system is built with responsive design in mind. Components adapt to different screen sizes based on the following breakpoints:

- Mobile: < 640px
- Tablet: 640px - 1024px
- Desktop: > 1024px

```jsx
// Example of responsive styling with Tailwind
<div className="
  text-sm md:text-base lg:text-lg
  p-2 md:p-4 lg:p-6
">
  Responsive content
</div>
```

## Accessibility

All components are designed with accessibility in mind. They follow WCAG 2.1 AA standards, which include:

- Proper color contrast
- Keyboard navigation
- ARIA attributes
- Focus management
- Screen reader compatibility

## Best Practices

1. **Use design tokens consistently**
   - Stick to the predefined color scales, typography, and spacing
   - Avoid hardcoded values in components

2. **Prefer composition over props**
   - Use multiple components together rather than creating complex, configurable components

3. **Use the right component for the job**
   - Don't repurpose components for unintended uses

4. **Maintain responsive design**
   - Test components at all breakpoints
   - Use responsive utilities consistently

5. **Keep accessibility in mind**
   - Ensure sufficient color contrast
   - Add appropriate ARIA attributes
   - Test with keyboard navigation
